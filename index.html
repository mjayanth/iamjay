The Three-Layer Stack for Production Robotics

How to architect autonomous robotic systems that actually ship to productionâ€”not demos that work 80% of the time, but systems achieving 99.9% uptime under real-world constraints.

Layer 1: Perception & Sensor Fusion (10-100ms latency budget)

The foundation layer processes multi-modal sensor streamsâ€”LiDAR point clouds (30 FPS @ 1.2M points), stereo vision (60 FPS @ 1920x1080), IMU data (1000 Hz), and proprioceptive feedback from joint encoders. Critical insight: don't fuse at the raw data level. Instead, run parallel perception pipelines and fuse at the semantic layer.

ROS2 Humble CUDA 12.1 TensorRT 8.6 ZED SDK Ouster LiDAR

We achieve 47ms end-to-end latency by partitioning models: lightweight object detection (YOLOv8-nano) runs on edge TPUs, while heavier scene understanding (SAM + CLIP) runs on datacenter GPUs, with predictions cached and interpolated during the round-trip.

Layer 2: Planning & Decision-Making (100-500ms budget)

This is where classical robotics meets modern AI. Hybrid architecture combining:

  • Model Predictive Control (MPC) for trajectory optimizationâ€”solving constrained optimization problems 20x per second using CasADi + IPOPT solvers
  • Reinforcement Learning policies (PPO-trained) for high-level task selection and recovery behaviors
  • LLM-based reasoning (Sonnet 4.5 via MCP) for novel scenario handling and human instruction interpretation

Key architectural decision: keep LLMs out of the control loop. They advise the planner but never directly command actuators. This prevents non-deterministic behavior in safety-critical paths while preserving adaptability.

The execution layer runs on real-time Linux (PREEMPT_RT kernel) with SCHED_FIFO scheduling. Control loops execute at 1kHz with deterministic 800Î¼s worst-case latency. Safety monitors implement three-tier protection:

Systems Analysis

Why Vision-Language-Action Models Will Replace Classical Control

The robotics industry is experiencing a Cambrian explosion in VLA (Vision-Language-Action) models. Here's the technical case for why they represent a paradigm shift, not just incremental improvement.

The Brittleness Problem in Classical Control

Traditional robotic control stacks require hand-engineering for every scenario. When a warehouse robot encounters an overturned box, you need human engineers to:

  • Write new perception rules for detecting "box in unusual orientation"
  • Define grasping heuristics for non-standard poses
  • Update motion planners to accommodate new manipulation primitives
  • Deploy, test, repeatâ€”often taking 6-8 weeks per edge case

This is fundamentally unscalable. The long tail of edge cases is infinite.

VLAs: Learning Visuomotor Policies End-to-End

Vision-Language-Action models (RT-2, OpenVLA, Ï€â‚€) directly map from pixels + language to robot actions. Architecture: transformer backbone (typically 1B-7B parameters) trained on millions of robot trajectories, learning a unified policy across manipulation, navigation, and interaction tasks.

OpenVLA-7B RT-2 Architecture RLDS Dataset Diffusion Policies

Key insight: VLAs exhibit emergent generalization. A model trained on "pick up a cup" generalizes to "pick up the overturned cup" without additional engineering. The language conditioning provides zero-shot task specification.

The Data Scaling Law

VLAs follow scaling laws similar to LLMs. Preliminary research shows:

  • Success rate improves log-linearly with dataset size (currently ~10M trajectories)
  • Performance gains continue beyond 1B parameters (no saturation yet observed)
  • Compute-optimal training requires ~2000 GPU-hours per 1M trajectories for 7B models

Implication: whoever aggregates the most diverse robotic training data wins. This favors companies with large-scale deployment fleets (Tesla, Figure AI) or simulation infrastructure (Google DeepMind).

The Economics Shift

Classical robotics: high capex (hardware), low opex (deterministic software)
VLA robotics: moderate capex + continuous opex (data collection, model retraining, GPU inference)

This transforms robotics from a hardware business to a software-as-a-service model. You're not selling robotsâ€”you're selling continuously improving autonomy subscriptions. First movers who build data flywheels will compound advantages indefinitely.
